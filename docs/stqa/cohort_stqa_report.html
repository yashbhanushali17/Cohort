Project Profile

Project Title

Cohort - Real-Time Social Communication Platform

Project Type

Full-stack web application

Objective

To create a unified platform for messaging, communities, events, updates, and AI interaction.

Target Users

Students and general users who need social communication and collaboration features in one app.

Main Modules

  • Authentication and onboarding
  • Direct and group chat
  • Communities and groups
  • Events and expiring updates
  • AI assistant and translation
Project ProfilePage 8

Existing System

Observed Limitations

  • Users often depend on different apps for chat, updates, and event coordination.
  • Traditional academic messaging projects lack advanced realtime behavior.
  • AI support and translation are usually absent.

Need for Improvement

  • Single platform for communication and coordination.
  • Better social discovery and group management.
  • Modern user experience with live updates and richer interaction.
Existing SystemPage 9

Proposed System

Cohort is proposed as an integrated communication platform where the same user can manage contacts, chat in realtime, participate in communities, create events, post updates, and use AI assistance without leaving the application ecosystem.

Realtime

Typing indicators, read receipts, online status, and socket-based updates.

Social

Friends, communities, groups, event participation, and personal updates.

Intelligent

AI chat and multilingual translation for better accessibility.

The proposed system improves both usability and technical structure by separating UI, APIs, realtime events, and data storage.

System Flow Diagram

User
  |
  v
React Frontend
  |---- REST APIs ----> Express Backend ----> MongoDB Atlas
  |
  |---- Socket.IO ----> Realtime Server
  |
  |---- AI Request ---> Gemini API

Flow Summary:
1. User performs action in frontend.
2. Frontend sends API requests to backend.
3. Backend stores / fetches data from MongoDB.
4. Realtime events are delivered through Socket.IO rooms.
5. AI requests are forwarded to Gemini and returned to UI.

Conclusion

The Cohort project demonstrates a complete full-stack communication platform that combines realtime messaging, social organization, events, updates, and AI assistance into a single system.

From a learning perspective, the project provided practical exposure to React-based UI development, Node.js / Express API design, Socket.IO realtime workflows, MongoDB schema design, authentication, and documentation.

The architecture is modular enough for future improvements such as automated tests, deployment hardening, media enhancements, and stronger production call support.
